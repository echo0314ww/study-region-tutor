# 2026-05-12 实施记录

## 背景

本次改进围绕四个方向展开：

- 一键诊断：让用户看到详细诊断结果、失败原因和修复建议。
- 设置向导：版本更新后首次打开出现整体功能向导，并为本版本新增向导预留接口；两类向导都可跳过，也可在设置页重新打开。
- 结果导出：支持复制和导出当前题目的 Markdown 学习记录。
- 代理服务多 Token + 限流：支持给不同用户分发独立 Token，并按 Token 配置内存级限流。

## 实施内容

### 一键诊断

- 新增共享类型 `DiagnosticStep`、`DiagnosticResult` 和 `RunDiagnosticsRequest`。
- 新增 IPC 通道 `tutor:run-diagnostics`，并在 preload 中暴露 `window.studyTutor.runDiagnostics`。
- 新增 `src/main/diagnostics.ts`，按当前连接模式执行诊断。
- 本地直连诊断包括：
  - 用户配置文件是否存在。
  - API 服务商是否读取成功。
  - API Key 是否存在。
  - `/models` 是否可用。
  - 当前模型是否已选择。
- 代理模式诊断包括：
  - 代理 `/health` 是否可用。
  - Token 是否能拉取 `/providers`。
  - 当前代理服务商是否可用。
  - `/models` 是否可用。
  - 当前模型是否已选择。
- 设置页新增诊断区域，诊断项以 `pass`、`warn`、`fail` 展示，并包含“可能原因”“处理建议”和脱敏技术细节。

### 设置向导

- 新增 `GuideKind`、`GuideDefinition`、`GuideStep` 类型。
- 新增 `src/renderer/src/guides.ts`，当前包含整体功能向导内容，并预留本版本新增向导定义。
- 新增 `GuidePanel` 组件。
- 应用版本变化后，如果当前版本还没有标记为已查看，会自动打开整体功能向导。
- 设置页顶部新增“整体功能向导”和“本版本新增向导”按钮，可随时重新查看。
- 当前版本新增向导暂为空态，但接口和 UI 已保留。

### 结果导出

- 新增共享格式化模块 `src/shared/exportConversation.ts`。
- 新增 IPC 通道 `tutor:export-conversation`，主进程通过保存对话框写入 Markdown 文件。
- 结果面板新增 `复制 Markdown` 和 `导出 Markdown`。
- 导出内容包含应用版本、导出时间、模型、输入方式、输出语言、只讲思路设置、首次讲解和追问记录。
- 默认不导出截图、API Key、代理 Token 或代理服务地址。
- 新增 `tests/exportConversation.test.ts` 覆盖导出格式和隐私说明。

### 代理服务多 Token 与限流

- `server/proxy-server.mjs` 继续兼容旧 `TUTOR_PROXY_TOKEN`，作为 `default` Token。
- 新增具名 Token 配置：
  - `TUTOR_PROXY_TOKENS=student_a,student_b`
  - `TUTOR_PROXY_TOKEN_STUDENT_A=...`
  - `TUTOR_PROXY_TOKEN_STUDENT_B=...`
- 新增可选限流配置：
  - `TUTOR_PROXY_RATE_LIMIT_PER_MINUTE`
  - `TUTOR_PROXY_RATE_LIMIT_BURST`
  - `TUTOR_PROXY_RATE_LIMIT_<TOKEN_ID>_PER_MINUTE`
  - `TUTOR_PROXY_RATE_LIMIT_<TOKEN_ID>_BURST`
- 限流按 `token id + endpoint` 使用内存级 token bucket。
- 429 响应会返回 `Retry-After` 和不含 Token 明文的错误信息。
- `/health` 增加 `tokenCount`、`rateLimitEnabled` 和 `providerCount`，便于诊断。

## 后续建议

- 后续版本可以在 `releaseGuide(version)` 中填充“本版本新增向导”的真实步骤。
- 一键诊断当前默认不跑深度文本请求，避免消耗用户额度；后续可在 UI 上增加“深度测试”开关。
- 如果代理服务需要跨进程或长期额度统计，内存级限流可以升级为文件或 Redis 等持久化存储。
