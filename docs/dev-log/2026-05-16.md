# 2026-05-16 实施记录：v1.2.0 发布准备

## 背景

本次按统一发布要求把前期完成的学习库、复习、导出、模型评测、OCR 预处理和安全边界检查能力归档为 `1.2.0`，并补齐发布材料、应用内向导和发布前校验链路。

## 决策

- 版本号更新为 `1.2.0`，但不在本机手动发布 GitHub Release，仍由推送 `v1.2.0` tag 触发 GitHub Actions。
- 学习库新增能力统一写入本版本新增向导；历史版本新增向导继续保留旧版本条目。
- 新增 `npm run security:check` 作为发布前和 GitHub Actions 的固定校验项，覆盖 API Key、代理 Token、公告、导出和渲染层存储边界。
- 批量导出只包含文字、复习状态和结构化学习信息，不包含截图或敏感配置。

## 用户可见变化

- 学习库支持待复习、错题筛选、复习统计、答题反馈和下次复习安排。
- 讲解完成后异步提取学科、知识点、题型、难度、标签、关键点和易错点。
- 学习记录可批量导出为 Markdown、Anki CSV 或 Obsidian Markdown。
- 设置页新增模型评测，可比较模型和 Prompt 模板的耗时、输出长度、状态和主观评分。
- OCR 可选择不同预处理模式，提高复杂截图的识别可调性。
- 诊断和发布流程加入安全边界检查。

## 后续风险

- Windows Codex 环境中 `npm run test`、`npm run build`、`npm run validate` 偶尔会因 esbuild `spawn EPERM` 失败；如普通权限失败，需要按权限要求提升后重跑。
- 推送 tag 需要网络和仓库权限；如果本地环境无法推送，应保留本地 release commit/tag，并由有权限的环境执行推送。

## Unreleased 后续维护记录

### CI、学习库概览、IPC 校验和答案渲染安全

本轮只落实未发布改动，不进行版本发布、不更新版本号、不创建 tag。

- 新增 `.github/workflows/ci.yml`，在 PR 和 `main` 推送时运行文档检查、类型检查、Lint、测试、安全边界检查和构建。
- 新增 `.github/dependabot.yml`，按周检查 npm 和 GitHub Actions 依赖更新。
- 渲染层自动更新状态订阅拆出为 `useUpdateStatus`，作为继续拆分 `App.tsx` 的第一步；后续又补充 `getUpdateStatus`，渲染层启动时会主动读取当前更新状态。
- 学习库新增掌握率、近 7 天复习、学科分布、高频知识点和常见易错点概览，并补充统计测试；后续补充主存储损坏时回退旧历史、待处理/复习文案、当前筛选导出提示和标题/标签/易错原因 debounce 保存。
- 关键 IPC 请求新增 `src/shared/validators.ts` runtime 校验，覆盖截图讲解、区域 OCR、OCR 文本讲解、追问、模型列表、诊断、结构化学习信息提取、模型 / Prompt 评测、取消请求、结束本题、代理 Token 保存、公告/代理 URL、单题导出和学习库批量导出。
- 模型 / Prompt 评测增加前端组合数限制和停止评测能力，一次最多运行 20 组，取消后保留已完成结果。
- 答案渲染补充安全收紧：普通 HTML 样式文本继续由 React 转义；KaTeX 输出会移除辅助 annotation 中的原始 TeX，公式 tooltip 遇到 `javascript:`、`data:`、`vbscript:` 等危险协议时降级显示。
- 新增 `tests/answerRenderer.test.ts`，覆盖 HTML 样式文本转义、KaTeX 危险协议和 annotation 移除。
