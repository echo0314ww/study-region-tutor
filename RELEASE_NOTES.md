# Release Notes

这里记录面向用户展示的简要更新说明，可用于 GitHub Releases 页面。更详细的开发记录请看 `CHANGELOG.md`。

## Unreleased

暂无。

## v1.3.0

- 新增暗色模式，支持亮色、暗色和跟随系统切换；所有界面元素均已适配暗色主题。
- 新增中英文 i18n 基础设施；当前语言设置主要控制回答和导出语言，界面文案将逐步接入英文翻译。
- 新增错题统计仪表盘，展示统计卡片、学科雷达图、知识点和易错点柱形图，帮助直观了解学习薄弱方向。
- 新增学习数据备份与恢复，可将全部学习记录导出为 JSON 文件，也可从备份文件导入并选择替换或合并模式。
- 学习库新增复习概览，可查看掌握率、近 7 天复习、学科分布、高频知识点和常见易错点。
- 修复多个已知问题：评测面板模型名不同步、OCR 图片缩放异常、对话加载后不滚动到底部等。
- 安全加固：SSRF 防护补充 IPv6 地址检测，代理上游请求添加超时，密钥脱敏范围扩大。
- 无障碍改进：键盘焦点可见、焦点陷阱、减少动画支持、危险操作按钮样式区分。
- 学习库导出和编辑体验做了细化：明确导出当前筛选结果，输入编辑会减少频繁保存，本地学习库数据异常时也会尝试回退旧历史。
- 本版本还补充了合入前自动检查、脚本语法检查、请求校验、窗口安全加固、答案渲染安全维护和模型评测取消等改进。

## v1.2.0

- 学习库升级为可复习的错题本，支持今日待复习、错题筛选、复习反馈、下次复习时间和易错原因记录。
- 讲解完成后会自动提取知识点、题型、难度、关键点和易错点，帮助后续搜索和归类。
- 学习库可以批量导出为 Markdown、Anki CSV 或 Obsidian Markdown，方便整理到常用学习工具。
- 设置页新增模型 / Prompt 对比评测，可用同一道题比较不同模型和讲解模板的效果。
- OCR 新增预处理模式，可按截图情况选择自动增强、对比度、二值化或多路增强。
- 发布前新增安全边界检查，进一步确认敏感配置不会进入前端存储、诊断报告或导出文件。

## v1.1.3

- 新增首次配置向导、代理管理面板和可自定义快捷键，首次上手与日常操作更集中。
- 新增学习库/错题本、OCR 多候选确认、诊断深度测试，便于搜索、收藏、标记复习状态并排查模型链路。
- 新增 Provider 配置生成器、Prompt 模板和公告分组，后续扩展服务商、讲解风格和公告管理更规范。
- 文档体系调整为“统一接手入口 + 专题文档 + 自动校验”，新对话或新账号接手项目时更容易找到当前状态、发布流程和约束。

## v1.1.2

- 补充本版本新增向导，并新增历史版本向导回顾；直接安装最新版的用户也可以从设置页按版本补看以往新增功能。
- 代理服务会立即反映 `.env.local` 里的服务商配置变化；配置无效时会显示配置错误，不再继续使用旧的 provider。
- 代理和 ngrok 的环境配置优先级保持一致，端口变化时会自动协调本地代理与公网隧道。
- 跨显示器框选截图会合成为完整图片，减少双屏或负坐标副屏下识别内容缺失。
- 常用非敏感设置会在重启后保留；API Key 和代理 Token 不会写入普通 localStorage。
- 连续使用 OCR 时会复用语言 worker，第二次及后续识别通常更快。

## v1.1.1

- 复制/导出入口改名为“复制答案 / 导出答案”，并移动到回答窗口底部操作区。
- 第三方 API 兼容范围扩展到 Gemini 原生和 Claude / Anthropic 原生协议，Makelove 等聚合服务商可按协议类型配置。
- 思考程度会按当前服务商和模型显示可用档位：Claude Opus 4.6 支持 `max`，Gemini 会自动使用对应的 thinking 参数。
- 代理服务补充多 Token 和限流配置示例，方便给不同用户分配独立访问 Token。

## v1.1.0

- 设置页新增“一键诊断”，会给出详细的失败原因、处理建议和可复制的脱敏诊断报告。
- 新增整体功能向导：版本更新后首次打开会显示，也可以在设置页顶部重新查看；本版本新增向导入口已预留。
- 结果面板新增复制/导出 Markdown，可保存当前题目的讲解和追问记录，默认不导出截图或敏感配置。
- 代理服务支持多个具名 Token，并可按 Token 配置限流；旧版单 Token 配置仍然兼容。
- 文档管理和发布检查流程更完整，新增 Codex 接手指南、架构说明、发布清单和文档自检；GitHub Actions 发布前也会检查文档和公告一致性。

## v1.0.3

- 顶部“截图”改为拖拽截图模式：点击后按住拖出题目区域，松开后先确认，点击“确认识别”后才会提交识别；右键或 Esc 可取消。
- 点击“截图下一题”或“结束本题”后会先隐藏结果窗口并进入截图模式，下一题确认识别后再显示结果窗口，减少框选时的遮挡。
- 修复启动后需要先点一次工具栏，透明区域外部点击才会生效的问题。
- 回答内容开始出现时会立即隐藏“处理过程”，减少阅读时的干扰。
- 结果内容较长时会显示“跳到底部”按钮，方便直接查看回答末尾结论。
- 本题追问中的用户消息改为右对齐，更接近对话模式。

## v1.0.2

- OCR 识别后会先进入“结果确认”页面，可在发送讲解前修改识别文本，减少截图模糊或公式识别错误带来的误答。
- 直接发送图片失败时，会转为本地 OCR 并让用户确认文本，不再自动把可能有误的 OCR 内容发给模型。
- 数学公式显示升级为 KaTeX 渲染，分数、根号、上下标等公式会更接近标准排版。
- 即使模型偶尔输出 `x²/16 + y²/12 = 1`、`4√7/7` 这类普通文本公式，应用也会尽量转换成 LaTeX 后按公式块显示。
- 回答中的 `LaTeX:` / `补充 LaTeX:` 标签会被隐藏，重复的扁平公式文本会被自动去掉，默认只展示渲染后的公式。
- 顶部工具栏和设置面板支持拖动，本地直连配置指引会显示实际配置文件路径。
- 主界面内部已拆成独立面板和 hooks，现有功能不变，后续继续加功能会更稳。

## v1.0.1

- 本地直连配置改为固定读取 `%APPDATA%\study-region-tutor\.env.local`，用户配置位置更明确。
- 本地直连失败时，设置页会显示配置指引并隐藏后续高级选项，避免展示技术错误。

## v1.0.0

- 顶部工具栏新增退出应用按钮，点击并确认后即可关闭应用。
- 新增版本更新公告文件，公告面板可以同时展示版本更新公告和私人公告。
- 发布流程使用 GitHub Actions 自带的 `GITHUB_TOKEN`，不需要额外配置 Personal Access Token。
- 版本更新公告默认折叠显示，点击标题后再展开具体内容。
- 公告不会自动弹出；公告按钮红点改为根据公告内容哈希判断，内容不变时不会重新提示。
- 公告级别改为任意文本标签，留空时只显示发布时间。
- 代理访问 Token 首次填写成功后会保存在本机，后续打开应用可直接使用；如果服务端 Token 已更换，应用会提示重新填写。
- 检查更新不再自动下载更新包；发现新版本后，需要点击“立即更新”才会开始下载，下载完成后再点击“重启安装”。

## v0.5.0

- 新增 ngrok 托管脚本，可自动启动公网隧道并回写公网代理地址。
- 代理服务现在会在 `/health` 中返回本机、局域网和公网访问地址，方便确认连接入口。
- 代理模式默认隐藏远程服务地址，只显示默认代理地址连接状态。
- 新增独立的高级设置调试视图，用于验证代理地址、填写自定义地址和恢复默认地址。
- 公告连接更安静，默认代理离线时会后台重试，不再启动时连续弹出公告请求错误。
- 设置面板进一步精简，不再展示不必要的 API Base URL 和 API Key 明细。

## v0.4.0

- 新增公告功能，应用可以接收并显示远程公告。
- 启动界面更轻量，默认只显示顶部工具栏；截图框、对话窗口和设置窗口可按需打开。
- 顶部工具栏新增公告入口，并支持未读提示和自动弹出。
- 公告内容支持按指定 ID 展示，便于只发布部分公告。
- 公告接口不需要 Token；API 代理功能仍然需要 Token。

## v0.3.1

- 优化悬浮窗口体验，识别和讲解过程中透明区域不再阻挡底层网页或桌面点击。
- 结果窗口、设置窗口和顶部工具栏仍可正常拖动、缩放、滚动和输入。
- 截图裁剪、多显示器和缩放比例处理保持不变，仍然只截取框选区域。

## v0.3.0

- 新增代理服务模式，可通过开发者本机代理服务使用第三方 API。
- 用户端只需要配置代理地址和访问 Token，不需要保存第三方 API Key。
- 代理模式支持刷新 API 服务商、刷新模型列表、图片讲解、OCR 文字讲解和本题追问。
- 截图和 OCR 仍在用户电脑本地完成，代理服务只处理用户主动发起的请求。

## v0.2.0

- 支持配置多个第三方 API 服务商。
- 设置面板新增 API 服务商选择，可在运行时切换使用的 API。
- 切换服务商后会自动重新加载对应模型列表。
- 请求失败时会提示当前服务商，方便切换到其他 API 后重试。
- API Key 只在主进程读取和使用，不会明文显示到界面。

## v0.1.0

- 初始版本，提供框选屏幕区域、截图识别和学习讲解的基础能力。
