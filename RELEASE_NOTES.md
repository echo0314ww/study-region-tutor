# Release Notes

这里记录面向用户展示的简要更新说明，可用于 GitHub Releases 页面。更详细的开发记录请看 `CHANGELOG.md`。

## Unreleased

暂无。

## v1.0.1

- 本地直连配置改为固定读取 `%APPDATA%\study-region-tutor\.env.local`，用户配置位置更明确。
- 本地直连失败时，设置页会显示配置指引并隐藏后续高级选项，避免展示技术错误。

## v1.0.0

- 顶部工具栏新增退出应用按钮，点击并确认后即可关闭应用。
- 新增版本更新公告文件，公告面板可以同时展示版本更新公告和私人公告。
- 发布流程使用 GitHub Actions 自带的 `GITHUB_TOKEN`，不需要额外配置 Personal Access Token。
- 版本更新公告默认折叠显示，点击标题后再展开具体内容。
- 公告不会自动弹出；公告按钮红点改为根据公告内容哈希判断，内容不变时不会重新提示。
- 公告级别改为任意文本标签，留空时只显示发布时间。
- 代理访问 Token 首次填写成功后会保存在本机，后续打开应用可直接使用；如果服务端 Token 已更换，应用会提示重新填写。
- 检查更新不再自动下载更新包；发现新版本后，需要点击“立即更新”才会开始下载，下载完成后再点击“重启安装”。

## v0.5.0

- 新增 ngrok 托管脚本，可自动启动公网隧道并回写公网代理地址。
- 代理服务现在会在 `/health` 中返回本机、局域网和公网访问地址，方便确认连接入口。
- 代理模式默认隐藏远程服务地址，只显示默认代理地址连接状态。
- 新增独立的高级设置调试视图，用于验证代理地址、填写自定义地址和恢复默认地址。
- 公告连接更安静，默认代理离线时会后台重试，不再启动时连续弹出公告请求错误。
- 设置面板进一步精简，不再展示不必要的 API Base URL 和 API Key 明细。

## v0.4.0

- 新增公告功能，应用可以接收并显示远程公告。
- 启动界面更轻量，默认只显示顶部工具栏；截图框、对话窗口和设置窗口可按需打开。
- 顶部工具栏新增公告入口，并支持未读提示和自动弹出。
- 公告内容支持按指定 ID 展示，便于只发布部分公告。
- 公告接口不需要 Token；API 代理功能仍然需要 Token。

## v0.3.1

- 优化悬浮窗口体验，识别和讲解过程中透明区域不再阻挡底层网页或桌面点击。
- 结果窗口、设置窗口和顶部工具栏仍可正常拖动、缩放、滚动和输入。
- 截图裁剪、多显示器和缩放比例处理保持不变，仍然只截取框选区域。

## v0.3.0

- 新增代理服务模式，可通过开发者本机代理服务使用第三方 API。
- 用户端只需要配置代理地址和访问 Token，不需要保存第三方 API Key。
- 代理模式支持刷新 API 服务商、刷新模型列表、图片讲解、OCR 文字讲解和本题追问。
- 截图和 OCR 仍在用户电脑本地完成，代理服务只处理用户主动发起的请求。

## v0.2.0

- 支持配置多个第三方 API 服务商。
- 设置面板新增 API 服务商选择，可在运行时切换使用的 API。
- 切换服务商后会自动重新加载对应模型列表。
- 请求失败时会提示当前服务商，方便切换到其他 API 后重试。
- API Key 只在主进程读取和使用，不会明文显示到界面。

## v0.1.0

- 初始版本，提供框选屏幕区域、截图识别和学习讲解的基础能力。
